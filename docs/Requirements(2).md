# Введение {#введение}

Целью проекта является разработка Telegram-бота, который помогает пользователям следить за рынком криптовалют и принимать инвестиционные решения.

## Основные функции {#основные-функции}

1. Настройка предпочтений: выбор криптовалюты по умолчанию.  
2. Настройка предпочтений: выбор фиатной валюты по умолчанию.  
3. Отображение текущей цены выбранной криптовалюты.  
4. Отображение истории цены выбранной криптовалюты за указанный период.  
5. Сравнение цен двух криптовалют.  
6. Добавление актива в портфель (с указанием количества).  
7. Удаление актива из портфеля (с указанием количества)  
8. Вычисление стоимости портфеля в данный момент времени.  
9. Отслеживание общей стоимости портфеля во времени.  
10. Расчёт и отображение изменения стоимости по каждой криптовалюте в портфеле.  
11. Получение от LLM инвестиционного анализа по конкретной криптовалюте.  
12. Получение от LLM обзора портфеля с рекомендациями.  
13. Запросы к LLM-советнику в свободной форме о крипторынке.  
14. Установка ценовых алертов на определенные криптовалюты.  
15.  Автоматические уведомления при изменении цены более, чем на 5% для отслеживаемых криптовалют.  
16.  Автоматические уведомления при превышении ценой установленного значения для криптовалюты из отслеживаемых валют.

# Функциональные требования {#функциональные-требования}

## Аутентификация пользователей {#аутентификация-пользователей}

Идентификация происходит по Chat ID, дополнительная регистрация не требуется.

## Настройки взаимодействия {#настройки-взаимодействия}

* Выбор пользователем фиатной валюты для представления цены криптовалюты:  
  ○  Если пользователь не указал фиатную валюту, добавлять фиатную валюту по умолчанию (доллары США).  
  ○  Представление цены криптовалюты в виде фиатной валюты устанавливается единым образом для всех активов, используемых для отслеживания.  
  ○  Выбор фиатной валюты предоставляется из заранее определенного списка, который включает евро, японскую иену, фунт стерлингов, турецкую лиру, российский рубль, китайский юань.   
  ○  Если данная валюта уже выбрана, выдавать соответствующее сообщение и прекратить процесс выбора фиатной валюты.  
* Просмотр текущей фиатной валюты для представления цены криптовалюты.  
* Выбор пользователем криптовалют для отслеживания  
  ○ Добавление криптовалюты в отслеживаемые  
- Если пользователь не добавил ни одну криптовалюту для отслеживания, то в отслеживаемых указывается криптовалюта по умолчанию (BTC).  
- Выбор криптовалют предоставляется из заранее определенного списка, который включает BTC, ETH, SOL, XRP, ADA, DOGE, AVAX, NEAR, LTC.  
-  Если добавляемая валюта уже находится в отслеживании, выдавать соответствующее сообщение.    
  ○ Удаление криптовалюты из отслеживаемых

               \-   Выбор криптовалюты для удаления предоставляется из списка отслеживаемых на данный момент криптовалют. 

               \-   Если после удаления в отслеживаемых не остается ни одной криптовалюты, то выдается соответствующее сообщение и в отслеживаемые добавляется криптовалюта по умолчанию (BTC).

* Просмотр текущего списка отслеживаемых криптовалют.  
* Просмотр ответа на вопрос в свободной форме LLM-консультанту о криптовалютных рынках.  
* Просмотр инвестиционного анализа на основе LLM для конкретной криптовалюты.

## Взаимодействие с рыночными данными {#взаимодействие-с-рыночными-данными}

* По запросу пользователя показывать текущую цену выбранной криптовалюты в ранее выбранной фиатной валюте.   
* По запросу пользователя показывать отчет об изменении цены выбранной криптовалют в фиатной валюте за указанный период :1 день, 7 дней, 30 дней.   
* По запросу пользователя сравнивать цены на две выбранные криптовалюты.

## Управление портфелем {#управление-портфелем}

* Добавление актива в портфель с указанием: тикера криптовалюты и количества монет  
* Удаление актива из портфеля с указанием: тикера и количества монет. Выбор актива для удаления предоставляется из списка текущих активов в портфеле.  
* Показывать содержимое портфеля с количеством монет и стоимости каждого актива.  
* Вычислять общую стоимость портфеля на данный момент  
* Отслеживать изменение стоимости портфеля за выбранный период: год, месяц, сутки  
* Показывать изменение стоимости каждой криптовалюты в портфеле с момента первого добавления валюты в портфель.  
* Получение обзора портфеля с рекомендациями LLM.

## История портфеля {#история-портфеля}

* Хранить следующую информацию о покупке актива:  
* Дата в формате YYYY-MM-DD HH:MM:SS.  
* Название криптовалюты.  
* Цена в валюте по умолчанию. 


## Уведомления {#уведомления}

Уведомления – сообщения, несущие информацию о изменениях в отслеживаемом активе, которые могут побудить пользователя к действиям на рынке (покупке или продаже).

●  	Посылать уведомления пользователю, когда:

○   Цена за единицу криптовалюты пересекает границу равную установленному пользователем значению для данной криптовалюты. 

○  	Цена криптовалюты изменяется более, чем на 5% вверх или вниз за последние 24 часа. Уведомления начинают присылаться автоматически при добавлении криптовалюты в отслеживаемые.

●  	Формировать и выводить список актуальных уведомлений для пользователя на данный момент.

●  	Удалять уведомления из списка отправленных при потере актуальности уведомлений, то есть по истечении срока в 7 дней.

# Нефункциональные требования {#нефункциональные-требования}

Производительность

* Запросы, которые зависят от внешних сервисов, должны выполняться неблокирующе: пользователь получает подтверждение приема запроса, а результат отправляется по мере готовности.  
* Оптимизация базы данных: Структура базы данных PostgreSQL должна проектироваться с использованием необходимых индексов (например, по chat\_id), чтобы обеспечивать быстрый поиск пользователей и их настроек без длительных блокировок.

Надежность

* Отказоустойчивость при работе с внешними API: При временной недоступности сторонних сервисов (API криптовалютной биржи Binance, LLM API, Telegram API) система должна использовать механизм повторных запросов (Retry) с возрастающей задержкой (например: 1с, 2с, 4с).  
* Гарантия доставки: В случае сетевого сбоя при отправке уведомления пользователю, задача не должна теряться. Сообщение должно возвращаться в очередь брокера сообщений для повторной попытки отправки.

Безопасность

* Все конфиденциальные данные (токен Telegram-бота, ключи LLM API, учетные данные базы данных и брокера сообщений) должны передаваться исключительно через защищенные переменные окружения. Хранение ключей в исходном коде запрещено.  
* Все данные, поступающие от пользователя (названия тикеров, числовые значения цен, объемы активов), должны проходить строгую валидацию по регулярным выражениям и диапазонам допустимых значений (например, отсутствие отрицательных сумм покупки).

Роли и доступы  
В системе предусмотрены две разделенные роли:

1. Пользователь (User): Клиент системы, взаимодействующий с ней через интерфейс Telegram-бота. Имеет доступ только к своим рыночным предпочтениям, личному портфелю, истории запросов к LLM и своим уведомлениям. Идентификация происходит по chat\_id.  
2. Администратор (Admin): Пользователь с привилегированными правами, взаимодействующий с системой напрямую через вызовы HTTP API. Имеет доступ к системным метрикам и обезличенным данным всех пользователей.

Требования к авторизации

* Авторизация через Telegram (Роль: User):  
  * Дополнительная регистрация с вводом логина/пароля не требуется.  
  * При первой отправке команды /start система извлекает уникальный chat\_id пользователя и создает новую запись в базе данных.  
  * Все последующие команды бота проверяют наличие данного chat\_id в системе. Если пользователя нет в БД, запрос отклоняется с предложением отправить /start.  
* Авторизация через HTTP (Роль: Admin):  
  * Осуществляется путем передачи токена доступа в HTTP-заголовке Authorization (формат: Bearer \<API\_TOKEN\>).  
  * Токен администратора должен быть надежно сгенерирован и задан на этапе развертывания через переменные окружения.

Конечные точки (HTTP Endpoints)

Сервер должен предоставлять следующие REST API эндпоинты для административного управления и мониторинга:

1\. Проверка работоспособности системы

* Метод и путь: GET /healthcheck  
* Авторизация: Не требуется (публичный доступ).  
* Описание: Эндпоинт для мониторинга состояния приложения.  
* Формат ответа: JSON-объект, содержащий текущий статус сервера (например, {"status": "UP"}) и массив с именами авторов-разработчиков проекта.

2\. Получение списка пользователей

* Метод и путь: GET /users  
* Авторизация: Строго требуется (только для роли Admin). Запросы без валидного заголовка Authorization должны получать статус 401 Unauthorized.  
* Описание: Возвращает информацию о зарегистрированных пользователях системы.  
* Формат ответа: Массив JSON-объектов, содержащий базовые данные пользователей (внутренний ID, дата регистрации, установленные валюты по умолчанию и т.д.).

* Логирование: SLF4J

# Глоссарий {#глоссарий}

## Основные термины {#основные-термины}

| Telegram  | кроссплатформенный мессенджер, в котором пользователи обмениваются сообщениями и взаимодействуют с ботами. |
| :---- | :---- |
| **Telegram-бот** | программное приложение, работающее внутри Telegram и автоматически обрабатывающее команды и сообщения пользователей. |
| **Chat ID** | уникальный числовой идентификатор пользователя или чата в Telegram, используемый системой для идентификации и аутентификации пользователя. |
| **Пользователь** | лицо, взаимодействующее с ботом через Telegram и использующее его функции для получения информации о криптовалютах, управления портфелем и настройки уведомлений. |
| **Администратор**  | пользователь с расширенными правами доступа, который взаимодействует с системой через HTTP API и имеет доступ к административным функциям. |
| **Криптовалюта**  | цифровой актив, использующий криптографические методы для обеспечения безопасности транзакций и учета владения. |
