1.  Требования к технологическому стеку
* Telegram API: Приложение должно быть интегрировано с Telegram API для обеспечения функциональности бота.
* LLM API: Приложение должно быть интегрировано с LLM API  для обеспечения функциональности на базе искусственного интеллекта (OpenRoute).
* Язык программирования: Java 25  
* Фреймворк: Spring 7 (ReactiveSpring (WebFlux), JOOQ, Spring Rest Docs)
* База данных: PostgreSQL 18.3
* Контейнеризации приложения: Docker, Docker Compose
* Логирование: SLF4J
* Тестирование: JUnit
________________


2.  Компонентная диаграмма
  

2.1. Сервер
Роль: центральный компонент системы
Функционал: 
* Принимает входящие запросы от Telegram-бота и (администратора, с end-point).
* Координирует взаимодействие между Telegram-ботом и остальными модулями системы.
2.2. Telegram-бот
Роль: пользовательский интерфейс
Функционал: 
* Принимает входящие запросы и сообщения от  пользователей через Telegram.
* Передает сообщения на сервер.
* Отображает результаты выполнения запросов.
2.3. User (пользователь Telegram)
Роль: конечный потребитель сервиса
Функционал: 
* Взаимодействует с системой через Telegram-бота: 
   * просмотр, создание и удаление уведомлений,
   * взаимодействие с портфелем,
   * взаимодействие с рыночными данными,
   * выполнение запросов к LLM
* Может проверять состояние системы через команду \healthcheck 
2.4. Администратор (Admin)
Роль: пользователь, управляющий системой
Функционал: 
* Осуществляет административное управление через HTTP API.
* Получает доступ к списку зарегистрированных пользователей.
* Может выполнить проверку состояния системы через команду /healthcheck.
2.4. Модуль уведомлений (AlertsHandingModule)
Роль: управление уведомлениями
Функционал: 
* Формирование уведомления при превышении пороговой цены.
* Формирование уведомления при превышении изменения процентов цены.
* Добавление уведомлений.
* Удаление уведомлений.
* Формирование списка актуальных уведомлений.
2.5. Модуль информации о криптовалюте (CryptoInformationModule)
Роль: вывод информации о текущей криптовалюте
Функционал: 
* Настройка криптовалюты и фиатной валюты по умолчанию.
* Добавление и удаление отслеживаемой криптовалюты.
* Получение цены текущей криптовалюты.
* Вычисление изменения цены криптовалюты за указанный период.
* Вычисление разницы в цене между несколькими выбранными криптовалютами.
* Выполнение инвестиционного анализа на основе LLM для конкретной криптовалюты.
2.6. Модуль управления портфелем (PortfolioManagementModule)
Роль: управление портфелем
Функционал: 
* Добавление актива в портфель.
* Вычисление общей цены всех активов в портфеле.
* Вычисление изменения цены конкретной криптовалюты с момента добавления криптовалюты в портфель.
* Вычисление изменения стоимости портфеля за выбранный период.
* Получение обзора портфеля с рекомендациями LLM.
2.7. Модуль обработки текстовых запросов (MessageHandlingModule)
Роль: обработка запросов в свободной форме
Функционал: 
* Обработка запроса в свободной форме и передача в LLM.
* Получение ответа от LLM на заданный запрос.
2.8. Модуль авторизации пользователя (AuthUserModule)
Роль: авторизация пользователя
Функционал: 
* Обработка регистрации новых пользователей.
* Поиск или сохранение данных пользователей в PostgreSQL.
2.9. Модуль HealthCheck (HealthCheckModule)
Роль: системный мониторинг
Функционал: 
* Проверка работоспособности системы (/healthcheck).
* Предоставление информации о статусе сервера, внешних API и базы данных, команде разработчиков.
2.10. Модуль списка пользователей (UserGetModule)
Роль: получение списка пользователей
Функционал: 
* Обработка запросов администратора на получение списка всех пользователей.
* Предоставление списка пользователей администратору.
2.11. Модуль авторизации администратора (AuthAdminModule)
Роль: авторизация администратора
Функционал: 
* Проверка HTTP заголовка Authorization: Bearer <API_TOKEN>.
* Валидация токена администратора из переменных окружения.
2.12. API Binance 
Роль: внешняя база данных для хранения информации о криптовалютах
Функционал: 
* Предоставление актуальных цен криптовалют через REST API.
* Поддержка фиатных валют из установленного списка  для конвертации.
2.13. API RabbitMQ
Роль: обработка потоков сообщений
Функционал: 
* Реализация логирования.
* Реализация асинхронной обработки поступивших сообщений и передача их на LLM.
* Гарантия доставки уведомлений.
2.14. API OpenRoute
Роль: внешний LLM-сервис для генерации инвестиционных рекомендаций
Функционал: 
* Генерация инвестиционного анализа конкретной криптовалюты.
* Анализ портфеля с рекомендациями.
* Ответы на вопросы пользователей в свободной форме.
2.15. База данных
Роль: хранение всех данных системы
Функционал: 
* Хранение данных пользователя.
* Хранение списка актуальных уведомлений пользователя.
* Хранение портфеля пользователя и его истории.
* Хранение выбранных по умолчанию криптовалюты и фиатной валюты. 
* Хранение истории запросов к LLM.
* Хранение истории уведомлений.
* Хранение цены криптовалюты за указанный период
